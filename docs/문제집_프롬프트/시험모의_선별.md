# 시험 모의 선별 프롬프트

`output/problem_book_final/` 문제 은행(3,325문항)에서 **실제 1회 시험 규모**로 모의 세트를 뽑을 때 사용한다.  
추출·형식 규칙: [`extraction_guide.md`](../extraction_guide.md)

## 출제자 품질 기준 (매 회차 필수)

모의고사 작업 시 **강사가 아니라 필기 출제자**로 행동한다. 목표는 “많이 풀게 하기”가 아니라 **실제 시험에서 판별력이 있는 80문항**을 만드는 것이다.

### 출제자가 지켜야 할 것

1. **정확성** — 지문·정답은 `problem_book_final`·수험서·`국가계약법`/`지방계약법`/`조달사업법` 체계와 **모순 없어야** 한다. 애매한 문항은 제외하거나 교체한다.
2. **시험자가 알아야 할 핵심** — 조달 실무·법제에서 반드시 구분·암기·판단해야 하는 내용 위주:
   - 입찰 방식·절차·보증금·해제/해지·낙성/요물·부정당업자
   - 발주계획·사전규격·시장조사·원가·협상·구매 전략
   - 계약 이행·검사·대금·변경·MAS·물가변동·지체상금·하도급
3. **단원(Part·Chapter) 골고루** — 한 단원·한 주제에 몰지 않는다. Intro `page_0002.jpg` **출제기준 (30)/(20) 표기** 비중을 참고해 각 Part에서 고르게 뽑는다.
4. **중복 금지** — 동일 지문·동일 주제(파레토·MAS 기준금액 등)·유사 쌍둥이 문항 금지 (`tools/select_mock_exam.py`의 `TOPIC_MAX`·`validate_round`).
5. **이전 회차와 겹치지 않기** — 〈K−1〉회차까지 쓴 지문은 〈K〉회차에서 제외.
6. **유형 균형** — `단원별 출제예상` 중심, ㉠㉡·빈칸형 일부 포함, Check Q&A는 과목당 **20% 이하**. O/X는 필기에 없으므로 **제외**.

### 과목별 출제 초점 (요약)

| 과목 | 문항 | 단원·핵심 |
|------|-----:|-----------|
| 1 공공조달과 법제도 | 30 | 공공조달 원칙·법령·입찰·계약 유형·전자조달·정책·윤리 |
| 2 계획 수립·분석 | 20 | 발주·수요분석·원가·협상·시장·구매전략·리스크 (Part 골고루) |
| 3 공공계약관리 | 30 | 물품·용역·공사 이행 / MAS / 보증·대금·변경·검사 (Part 1~4 분산) |

### 회차 완성 후 출제자 검수 체크리스트

- [ ] 80문항·과목별 30/20/30
- [ ] Part·Chapter 분포 표 (한 Part에 과도하게 몰리지 않았는지)
- [ ] 동일 주제 중복 0건 (`validate_round` 통과)
- [ ] 빈출 함정 주제(보증금 구분, MAS 2단계, 해제/해지, 발주 순서 등) **적절히** 포함
- [ ] `필기_모의_문제.md`·`필기_모의_정답.md`·`필기_응시.html` 갱신
- [ ] 채팅 **과목별 분할 게시** (아래 §「채팅 응시 흐름」)

### 채팅 응시 흐름 (〈K〉회차, 사용자 선호)

파일은 **80문항 한 세트**로 저장하되, **채팅에는 과목별로 나눠** 올린다. 답안 체크 부담을 줄이기 위함.

| 순서 | 트리거 | 채팅에 게시 | 문항 번호 |
|:---:|--------|-------------|----------|
| 1 | 「〈K〉회차 출제」「모의고사」 등 | **1과목 전체** | **1~30** |
| 2 | 사용자 「다음 문제」「2과목」 | **2과목 전체** | **31~50** |
| 3 | 사용자 「다음 문제」「3과목」 | **3과목 전체** | **51~80** |

**규칙**
- 각 배치마다 해당 과목 **문항 전부**를 채팅에 표시 (선지 줄바꿈).
- **정답·해설은 사용자가 해당 구간 답을 보내기 전까지 공개하지 않음.**
- 1과목만 풀고 답 제출 → 채점 후, 「다음 문제」 요청 시 2과목 게시.
- 80문항 완료 후 전체 채점·`필기_풀이.md` 작성 가능.
- 다음 배치 안내 문구 예: *「1~30번 풀고 답 보내주세요. 2과목은 ‘다음 문제’라고 하시면 31~50번 올려 드립니다.»*

## A. 필기 모의 (80문항 · 한 파일)

선별은 과목별로 진행할 수 있으나, **산출물은 과목 구분 없이 한 문제집**으로 합친다.  
문항 번호는 **1~80 연속** (1과목 1~30 · 2과목 31~50 · 3과목 51~80).

작업 시 **〈K〉=모의 회차**(예: 1회차). 과목별 선별 시 **〈N〉=과목번호**, **〈M〉=해당 과목 문항 수**만 교체한다.

### A-1. 과목별 선별 (에이전트 1개 = 1과목)

```
너는 공공조달관리사 1회 필기 〈N〉과목 **모의 출제자**다.
실제 Q-Net 필기처럼 **정확하고 판별력 있는** 문항만 고른다.
`output/problem_book_final/〈slug〉/parts_clean/part*.md`에서 **〈M〉문항**을 선별한다.
최종 병합 전까지는 과목 내 번호 1~〈M〉으로 작업한다.

## 출제자 원칙
- 시험자가 조달 업무·법제에서 **반드시 알아야 할** 개념·절차·구분 위주
- 각 Part·Chapter에서 **고르게** — 출제기준 (30)/(20) 비중 참고
- 정확한 지문만 — 수험서·법령과 어긋나면 제외
- 동일 주제·유사 지문 중복 금지 (파레토·MAS기준금액 등 회차당 1문항)
- 이전 회차 미출제 지문 우선

## 과목별 목표 문항 수
- 1과목: 30문항 → 병합 후 **1~30**
- 2과목: 20문항 → 병합 후 **31~50**
- 3과목: 30문항 → 병합 후 **51~80**

## 선별 우선순위 (높은 순)
1. `단원별 출제예상문제` — 4지선다
2. ㉠㉡·ㄱㄴㄷ **복수선택**·`( )` **빈칸**형 — Q-Net 예제 유형 (`sources/공식_qnet_예제문제/`)
3. `Check Q&A` — 개념 확인용 (전체의 20% 이하)
4. **제외**: `(O/X)` 문항 — 실제 필기에 없음

## 구성 원칙
- Part·Chapter 골고루 (한 Part에 〈M〉문항 몰지 않음)
- 1과목 Intro `page_0002.jpg` 출제기준 **(30)/(20) 표기** 비중 참고 (2과목은 20문항 분량에 맞게 핵심 단원 우선)
- 동일 페이지·유사 지문 중복 금지
- **동일 주제 중복 금지** — 파레토·MAS 2단계 기준금액 등은 **회차당 1문항** (`tools/select_mock_exam.py`의 `TOPIC_MAX` 참고)

## 출력 형식 (과목별 작업 중)
- 문항 번호 1~〈M〉 연속
- 선지 줄바꿈·㉠㉡ 소항목 분리는 extraction_guide 준수
- 각 문항 끝: `<!-- source: 원본 Part/페이지 -->` 유지

## 완료 보고
Part·Chapter별 문항 수, 제외한 OX 수, ㉠㉡·빈칸형 포함 개수.
```

## 선별 우선순위
1. `서술형 출제예상문제` — 10~12문항 (절차·이유·차이 설명)

## C. 검수 (모의 세트 완성 후)

1. 문항 수 = 목표 (1·3과목 30, 2·4과목 20)
2. OX 혼입 없음 (필기)
3. 선지·㉠㉡ 줄 분리 (`fix_choice_lines.py` 또는 수동)
4. 원본 source 주석 유지
5. **출제자 검수** — Part·Chapter 분포, 주제 중복, 빈출 핵심 포함 여부 (위 체크리스트)
6. (선택) Q-Net 예제 6문항·Intro 출제기준과 단원 분포 대조
