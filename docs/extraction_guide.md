본 저장소 문제집은 **박문각 수험서 전량 추출**(학습용 은행)이다. 실제 1회 시험과의 차이를 구분한다.

**통합 기준:** [`시험_안내.md`](시험_안내.md) — 시행공고(일정·방법) + Intro 출제기준(문항 수·합격 기준)

| 구분 | 시험 과목 | 문항 수 | 시험 방법 |
|------|-----------|--------:|-----------|
| 필기 | 공공조달과 법제도 이해 (1과목) | **30** | CBT 4지 택일형 |
| 필기 | 공공조달계획 수립 및 분석 (2과목) | **20** | 동일 · **필기 총 80문항 · 2시간** |

## 대상

- `sources/민간_박문각_수험서_jpg/<slug>/Part N/page_*.jpg` 원본 이미지
- OCR 보조: **1~4과목** (`output/ocr/<slug_with_underscore>/`, 예: `2과목_공공조달_계획분석`)
- 정확도 기준은 항상 원본 JPG. OCR은 보조·감사용
- 수험서 내부 `CHAPTER 01`, `CHAPTER 02` 등은 Part 안의 소단원이다.
- 1과목 `Intro` = 출제기준·목차 (문제 추출 제외)

## 포함할 문제 유형

- 이론 페이지 안의 `Check Q&A`, `Check O&A`, `V Check Q&A` 등 박스형 확인 문제
- `단원별 출제예상문제`
- `최종점검 OX 퀴즈`

## 제외할 내용

- `해설`, `정답`, `정답 및 해설`, 하단 정답표
- 손글씨 동그라미, 밑줄, 체크 표시 등 풀이 흔적
- 문제를 설명하는 이론 본문
- 표지, 목차, 학습목표, 키워드, 교재 활용 안내

## 정리 형식

### 객관식 (4지선다 등) — 필수

- **문항 줄**: `N. 문제문` (선지 없이 문제만)
- **선지 줄**: `①` `②` `③` `④` 각각 **별도 한 줄**, 앞에 공백 3칸
- **금지**: 선지를 문항 줄 끝이나 한 줄에 이어 쓰기

올바른 예:

```
8. 일반공사 DBB(Design-Bid-Build) 방식 설명으로 옳은 것은?
   ① 설계와 시공을 하나의 계약으로 일괄 발주한다.
   ② 설계 완료 후 시공입찰을 거쳐 분리 계약한다.
   ③ 시공사가 설계부터 운영까지 전담한다.
   ④ 공사관리자(CM)만 계약당사자가 된다.
```

잘못된 예 (절대 금지):

```
8. 일반공사 DBB...? ① ... ② ... ③ ... ④ ...
```

- 선지가 5개 이상(①~⑩)이어도 동일: **기호마다 한 줄**
- 서술형 소문항 `① … ② …`가 한 문항 안에 있을 때도 **각 소문항을 별도 줄**로 분리

### 복수선택·빈칸 (㉠㉡ / ㄱㄴㄷ)

- 소항목 `㉠` `㉡` `㉢` `㉣` 또는 `ㄱ.` `ㄴ.` `ㄷ.` `ㄹ.`은 **지문 다음 줄에 항목별 한 줄**
- 선지 `①~④`는 기존과 같이 **각각 한 줄** (3칸 들여쓰기)
- 빈칸 `( )`은 지문 안에 유지

올바른 예:

```
2. 다음 중 계약금액 조정 사유로 옳은 것을 모두 고른 것은?
   ㉠ 물가변동
   ㉡ 설계변경
   ㉢ 계약상대자의 단순 착오
   ㉣ 천재지변 등 불가항력적 사유
   ① ㉠, ㉡
   ② ㉠, ㉡, ㉣
   ③ ㉠, ㉡, ㉢, ㉣
   ④ ㉠, ㉡, ㉢
```

잘못된 예:

```
2. … 모두 고른 것은? ㉠물가변동 ㉡설계변경 ㉢… ㉣…
   ① ㉠, ㉡
```

### OX

- `N. 문제문 (O/X)` 한 줄

### 4과목 실기 — 암기팁 (학습 시 선택)

- 추출·빌드 파이프라인에서는 **암기팁을 일괄 추가하지 않음**
- 학습 중 에이전트에게 **그때그때** 필요하면 `> 💡 암기팁: …` 형식으로 요청 (source 직전)
- 정답·해설을 직접 밝히지 않음 (힌트·구분·숫자·절차만)
- 1~3과목 필기에는 암기팁 **작성하지 않음**

### 공통

- OCR이 애매하면 원본 이미지 기준 보정
- 각 문항 묶음 맨 아래: `<!-- source: Part N/page_XXXX.jpg -->`
