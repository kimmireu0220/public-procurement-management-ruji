# 시험 모의 풀이·해설 프롬프트

모의고사 **응시 후** 내 답을 제출했을 때 사용한다.  
출제 방법: [`시험모의_선별.md`](시험모의_선별.md)

## 역할 분리

| 단계 | 프롬프트 | 산출물 |
|------|----------|--------|
| 출제 | `시험모의_선별.md` | `필기_모의_문제.md`, `필기_모의_정답.md` |
| 풀이·해설 | **이 문서** | `필기_풀이.md`, `오답노트.md` 갱신 |

## 산출 위치

```
output/mock_exam/
├── README.md
├── 오답노트.md              ← 틀린 문항만 회차별 누적
└── 〈K〉회차/
    ├── 필기_모의_문제.md
    ├── 필기_모의_정답.md
    ├── 필기_풀이.md              ← 최초 응시 (시도 1)
    └── 풀이/                     ← 재응시(시도 2+)만
        └── 필기_시도2_YYYY-MM-DD.md
```

- 〈K〉=모의 회차 번호 — 경로·파일명에 `〈K〉회차`로 치환 (예: K=2 → `output/mock_exam/2회차/`)
- 문항 번호 **1~80 연속** (1과목 1~30 · 2과목 31~50 · 3과목 51~80)

## 입력 (내가 제공)
- 회차 (예: 〈K〉회차)

### 과목별 분할 응시·채점
- 1과목(1~30)만 제출 → 해당 구간만 채점·오답 요약 (정답지 전체 공개 금지)

- 80문항 모두 제출되면 `필기_풀이.md`·`오답노트.md` 전체 반영

## 참고 파일 (채팅에 첨부)
- `output/mock_exam/〈K〉회차/필기_모의_문제.md`
- `output/mock_exam/〈K〉회차/필기_모의_정답.md`
- 원문 근거: 문제지 각 문항의 `<!-- source: … -->` 및 `problem_book_final` 해당 페이지

## 채점 규칙
- 정답지 기준으로 O/X 판정
- 복수선택·㉠㉡형: 정답지와 동일해야 정답 (부분 점수 없음)
- 내 답 누락·형식 오류: 해당 문항 오답 처리 후 풀이에 명시
- 과목별 점수: 1과목(1~30), 2과목(31~50), 3과목(51~80) 각각 집계

## 해설 원칙
- **맞은 문항·틀린 문항 모두** 해설한다
- 맞음: 왜 정답인지 핵심 개념·근거 (2~4문장)
- 틀림: 정답 근거 + **오답 포인트** (내가 고른 선지가 왜 틀렸는지 1~2문장)
- 수험서·문제 원본 범위 안에서 설명. 불확실하면 "원본 확인 필요"
- 정답지의 한줄 요약을 그대로 복붙하지 말고 풀어서 설명

## B. 하지 말 것

## D. 오답 보강·응용 퀴즈 (사용자 선호 형식)

모의고사·보강 퀴즈 응시 **후** 틀린 주제를 짚을 때 사용한다.  
단순 암기 OX보다 **사례·판단형 응용**을 우선한다.

### 언제 내는가
- 회차 채점 후 오답 **한 줄 요약** 제공
- 사용자 요청 시 오답 주제 기준 **보강 N문항** (보통 7~12문항)
- 같은 주제를 맞출 때까지 **응용 변형** 추가 가능

### 문항 구성 원칙
1. **응용형** — 「A 기관이 ~했다」 「총액 물가변동 요건은 미충족인데 자재가 18% 상승」 등 **상황+판단**
2. **시험 함정 반영** — 보증금 종류 구분, MAS 기준금액, 파레토(건수 vs 금액), 합의해제·기성고, 단품슬라이딩(90일·15%) 등
3. **정확성** — `problem_book_final`·수험서와 모순 없는 선지만 사용
4. **중복 주제** — 한 세트 안에서 같은 개념을 **다른 각도**로 1~2회까지만

### 채팅 표시 형식 (필수)

```markdown
### 1번
지문 …

① 선지
② 선지
③ 선지
④ 선지

---

### 2번
…
```

- 문항 번호는 `### N번` + `---` 구분
- 선지는 **항목마다 줄바꿈**
- 하단에 답 입력 예시만: `예: 2 3 4 3 3 3 3`

### 정답·해설 공개 규칙
- **사용자가 답안을 보내기 전까지 정답표·해설·정답 행 열지 않음**
- 답안 수신 후: 문항별 O/X 표 + 틀린 것만 1~2문장 오답 포인트
- 미제출 문항은 「미응답」으로 표시하고, 나중에 이어서 채점

### 완료 보고 (답 제출 후)
- 점수 n/N
- 틀린 문항: 번호·한 줄 이유
- (선택) 추가 응용 3~7문항 제안 — 사용자가 원할 때만

## C. 사용자 요청 예시

- `〈K〉회차 필기 답 채점해줘: (80개 답)` 또는 `1~30번: …`
- `@output/mock_exam/〈K〉회차/필기_모의_문제.md 답 제출 — 시험모의_풀이.md 따라 풀이 기록해줘`
- `다음 문제` (2과목 31~50 또는 3과목 51~80 채팅 게시)
- `오늘 틀린 문제 한 줄 요약해줘`
- `틀린 부분 응용 문제 7개 내줘` (§D 형식·정답 비공개)
