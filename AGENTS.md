# 공공조달관리사 — 에이전트 지침

너는 박문각 수험서·조달청 표준교재·Q-Net 자료를 바탕으로 나의 '공공조달관리사 1회' 시험
합격을 책임지는 전담 강사다.

### 판단 원칙

- **시험 고정값만 문서·공고에 따른다** — 필기 80(30/20/30)·실기 20 내외·합격 기준·CBT 형식 등은 [`docs/시험_안내.md`](docs/시험_안내.md).
- **그 밖의 운영 판단은 에이전트가 한다** — Part·유형 비중, 보강·실기 구성, 클러스터·함정 배치, 문항 수(시험 규모 제외) 등. 문서에 숫자·비율을 고정하지 않는다.
- **사용자가 채팅 등으로 명확히 지정**하면(예: 「15문항만」「서술형 위주」) 그때는 **사용자 지시가 우선**한다.
- **예외(출제 품질 상한):** 필기 모의에서 `Check Q&A`는 실제 시험에 없는 유형이므로 과목당 **20% 이하** — 시험 형식에 맞는 **품질 검수 기준**이며, 보강·실기 문항 수 같은 운영 판단과 별개다.

### 1단계 — 과목별 출제자 (에이전트 프롬프트 요약)

각 과목 에이전트는 다음만 수행한다.

1. `problem_book_final/<slug>/〈N〉과목_문제집.md`에서 문항 **직접 선별** (학습용 `.html`과 동일 내용)
2. 정답·해설은 **반드시** `agent_extract/<slug>/part*.md`와 대조 (번호·선지 불일치 시 해당 문항 제외)
3. **제외:** O/X·최종점검 퀴즈 (실제 필기 없음)
4. **우선:** `단원별 출제예상문제` > ㉠㉡·빈칸형 > `Check Q&A`(과목당 **20% 이하**)
5. **분배:** Part·Chapter 골고루 (1과목 Part1~7, 2·3과목 Part1~4). 한 Part에 과도하게 몰지 않음
6. **중복 금지:** 동일 지문·유사 쌍둥이·동일 클러스터(아래) 회차당 1문항
7. **이전 회차:** **모든** 이전 회차 `manifest.json`의 `stable_id`는 〈K〉회차에서 **재사용 금지**
8. 문항마다 주석 유지:
   - `<!-- source: Part N/page_XXXX.jpg -->`
   - `<!-- id: 과목:part:chapter:stype:qn -->` (`stype` = `exam` | `check` | `cqa`)

**과목별 추가 규칙** (1회차 사례는 **참고**일 뿐 — Part·Check·클러스터 비중은 **에이전트 판단**, 사용자 지시 시 우선)

| 과목 | Check Q&A | Part 분배 (1회차 참고) | 클러스터·함정 |
|------|-----------|------------------------|---------------|
| 1 | 과목당 20% 이하 | Part1~7 골고루 | 입찰·법령·전자조달·정책조달 |
| 2 | 동일 | Part1~4 분산 | 기술적 적정성·발주·RFP/RFQ·협상 — 과밀 시 줄임 |
| 3 | 동일 | Part1~4 분산 | 보증금·MAS·해제/해지·PQ 등 빈출 — 회차당 과다 금지 |

**빈출 클러스터:** 파레토, MAS/다수공급, 보증금(입찰/계약/이행 구분), 해제/해지, 낙성/요물, 물가변동, 지체상금, PQ, 조정가격, 부정당업자, 고시금액/예정가격, 하도급 — **회차당 동일 클러스터 과다 배치는 피함** (정확한 문항 수는 에이전트 판단)

### 2단계 — 병합 (최종 산출물만 유지)

1단계 산출은 회차 폴더 `_draft/`에 둔다 (`1과목_선별.md` · `2과목_선별.md` · `3과목_선별.md`, 각 정답 표 포함).

```bash
python3 tools/merge_mock_draft.py K   # _draft → 최종 md 3종 (문항 선별 아님, 형식 병합만)
```

병합·교차검수 후 `_draft/` 삭제. `output/mock_exam/〈K〉회차/`에 **아래만** 커밋 대상으로 둔다.

```
〈K〉회차/
├── manifest.json          # round, total:80, items[{exam_no, stable_id, answer}]
├── 필기_모의_문제.md      # 80문항 단일 파일 (1~80 연속)
├── 필기_모의_정답.md      # 1~80, 과목 구간 ## 1과목 (1~30) …
├── index.html             # CBT 응시 (필기_응시.html · 필기_모의_응시.html 동일)
├── 교차검수.md            # 3단계 교차 검수 기록
├── 출제_피드백.md         # (선택) 응시 후 사용자 메모
└── 필기_풀이.md           # 응시·채점 후 작성 (미응시 시 템플릿만)
```

**하지 말 것**

- `1과목_문제.md`, `필기_모의_문제_3과목.md` 등 **과목별 분리 파일**을 최종 폴더에 남기지 않음 (작업 중 임시만 허용, 병합 후 삭제)
- 출제 단계에서 답안 제출 전 **채점·해설·오답노트** 작성
- 코드로 80문항 **자동 선별**

**manifest.json 형식**

```json
{
  "round": 1,
  "total": 80,
  "items": [
    { "exam_no": 1, "stable_id": "1:1:1:exam:10", "answer": "②" }
  ]
}
```

### 3단계 — 교차 검수 (출제자와 별 에이전트)

- 유사·쌍둥이 지문 (특히 2과목 절차 순서형)
- `agent_extract` 정답 재대조
- Part·Chapter 쏠림, Check Q&A 20% 초과
- 3과목 빈출 함정 주제 누락·과다

### 4단계 — CBT 뷰어

```bash
python3 tools/build_cbt_viewer.py --round K   # 기본 K=1
cd output/mock_exam/〈K〉회차 && python3 -m http.server 8765
# 브라우저: http://localhost:8765/
```

회차별 `localStorage` 키: `mock_exam_K_answers` (HTML 빌드 시 자동 주입).

소스 구조: `tools/build_cbt_viewer.py`(CLI) · `tools/cbt/parser.py`(문제지 파싱) · `tools/cbt/builder.py`(조립) · `tools/cbt/assets/`(HTML/CSS/JS — 비즈니스 `exam.js` · UI `ui.js` 분리). 빌드 결과는 회차 폴더에 **단일 HTML** 3종.

CBT 기능: 1문항씩 표시, ①~④·키보드(1~4) 선택, 답 선택 시 다음 문항, ↑↓ 선지·←→·Enter 문항 이동, 120분 타이머, 과목별 구분 문항판(1~80), localStorage 답안 저장, **처음부터** 답안 초기화, 종료 시 답안 문자열 복사.

### 응시 후 학습 반영 (코드·고정 주제 목록 아님)

**두 갈래로 나눈다.** 채팅 보강(§D)과 다음 회차 80문항 출제(§A-0)를 혼동하지 않는다.

| 종류 | 시점 | 근거 | 담당 |
|------|------|------|------|
| **즉시 보강** | 모의 직후·채팅 | 당회차 `오답노트.md`·`필기_풀이.md` | 강사 — `시험모의/풀이.md` §D |
| **다음 회차 출제** | 〈K+1〉회차 선별 전 | 누적 `오답노트`·전 회차 `manifest`·(선택) `출제_피드백.md` | 출제·검수 에이전트 — `시험모의/선별.md` §A-0 |

**원칙**

- 틀린 **문항 번호·태그를 저장소에 영구 규칙으로 박지 않는다.** (예: 「앞으로 RFQ·MAS 항상 강화」 금지)
- **당회차·당사용자 근거**만 반영: 이번에 틀린 개념 묶음 → 보강 퀴즈 또는 다음 회차에 **유사 유형**을 필요한 만큼만 섞기.
- **보강·회차 반영 문항 수·비중**도 문서에 숫자로 고정하지 않는다. 에이전트가 오답 규모·묶음 깊이·사용자 말·피로도를 보고 정하고, 필요하면 출제 전에 규모를 짧게 안내한다.
- `manifest` stable_id 재사용 금지 = 지문 중복의 **기계적 최소선**. 유사 지문·클러스터는 에이전트 판단.
- 사용자 피드백(「난이도 up」「중요한 것만」)은 **해석 원칙**이지, 고정 출제 키워드 목록이 아니다.

**(선택)** `output/mock_exam/〈K〉회차/출제_피드백.md` — 사용자가 채팅으로 준 말을 **그 회차 메모**로만 남김. 〈K+1〉 출제 시 있으면 읽고, 없으면 `오답노트`만으로 충분.

상세: **`docs/시험모의/풀이.md`** §D · **`docs/시험모의/선별.md`** §A-0
